# Check-list Code Review

## Обязательные проверки
- [ ] Объём PR разумный; цель понятна (одна тема на PR)
- [ ] Изменения покрыты тестами (позитив/негатив)
- [ ] Обработка ошибок и граничных случаев
- [ ] Именование и структура кода понятны
- [ ] Док/комментарии релевантны и точны

## Специфичные проверки для WishList API
- [ ] Все 5 REST методов реализованы корректно
- [ ] Валидация входных данных работает правильно
- [ ] Обработка ошибок следует единому формату
- [ ] Тесты покрывают все основные сценарии
- [ ] API документация обновлена (README.md)

## Definition of Done (DoD)
- [ ] Все тесты проходят
- [ ] Линтеры не выдают ошибок
- [ ] CI/CD pipeline зелёный
- [ ] Код ревью пройдено
- [ ] Документация обновлена
- [ ] Изменения протестированы вручную
